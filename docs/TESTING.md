# Тестирование

## 1. Команда запуска

```bash
python3 -m unittest discover -s tests -v
```

## 2. Что покрыто

Тестовый набор проверяет:

1. `add_matrices` — корректность поэлементного сложения;
2. `multiply_matrices` — корректность матричного произведения;
3. `determinant` — корректность вычисления определителя;
4. `rank` — корректность ранга для линейно зависимых строк;
5. `solve_slae_matrix_method`, `solve_slae_cramer`, `solve_slae_gauss` —
   согласованность всех методов на одной системе;
6. Негативный сценарий `solve_slae_cramer` при `det(A)=0`.

## 3. Политика верификации

- Любое изменение ядра должно сопровождаться тестами.
- При исправлении бага добавляется отдельный регрессионный тест.
- Перед коммитом рекомендуется полный прогон `unittest`.

## 4. Идеи для расширения покрытия

- Проверки численной устойчивости на матрицах большего размера;
- Набор случайных тестов с верификацией через NumPy (как oracle) в отдельном контуре;
- Тесты парсинга UI-слоя и сообщений об ошибках.
