# Матрица соответствия требованиям контрольной точки №1

Ниже перечислены требования, релевантные первой контрольной точке, и статус их реализации в проекте.

| Требование | Статус | Реализация |
|---|---|---|
| Операции над матрицами: сложение | ✅ | `add_matrices` |
| Операции над матрицами: вычитание | ✅ | `subtract_matrices` |
| Умножение матрицы на число | ✅ | `multiply_by_scalar` |
| Умножение матриц | ✅ | `multiply_matrices` |
| Транспонирование | ✅ | `transpose` |
| Нахождение определителя | ✅ | `determinant` |
| Нахождение ранга | ✅ | `rank` |
| Решение СЛАУ матричным методом | ✅ | `solve_slae_matrix_method` |
| Решение СЛАУ методом Крамера | ✅ | `solve_slae_cramer` |
| Решение СЛАУ методом Гаусса | ✅ | `solve_slae_gauss` |
| Ввод исходных данных пользователем | ✅ | Tkinter-поля ввода |
| Команда на выполнение операции | ✅ | Кнопки операций |
| Клавиатурный ввод | ✅ | `Text`/`Entry` поля |
| Ввод мышью | ✅ | Кнопки/фокус/вставка через GUI |
| Вывод промежуточных действий | ✅ | `CalculationResult.steps` в окне вывода |
| Вывод итогового результата | ✅ | `CalculationResult.result` в окне вывода |
| Сообщение о математической невозможности операции | ✅ | исключения `MatrixError` |
| Сообщение о сбое в ходе операции | ✅ | общий обработчик ошибок в UI |

## Примечания

1. Калькулятор реализован для функционального охвата первой контрольной точки.
2. Метрики типа «98% надежности» и «не более 10 секунд» требуют отдельного нагрузочного протокола и формальной методики измерений; базовые автотесты и практический запуск подтверждают корректную работу на типовых учебных данных.
