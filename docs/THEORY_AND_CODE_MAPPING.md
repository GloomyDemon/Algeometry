# Теоретическая документация вычислений с привязкой к коду

Документ объясняет **математическую теорию** каждой операции и показывает, **где именно** эта логика реализована в коде (`matrix_calculator.py`, `app.py`).

---

## 1. Базовые определения

### 1.1 Матрица

Матрица размера `m × n` — прямоугольная таблица чисел:

\[
A = (a_{ij}), \quad i = 1..m,\; j = 1..n
\]

В проекте матрица хранится как `List[List[float]]`.

**Код:** типы и контейнер результата объявлены в `matrix_calculator.py` (`Matrix`, `Vector`, `CalculationResult`).

### 1.2 Корректность формы (shape)

Для матричных операций требуется, чтобы все строки имели одинаковую длину.

**Теория:** матрица должна быть прямоугольной.

**Код:** `_shape(matrix)` проверяет непустоту и одинаковую длину строк; в противном случае бросает `MatrixError`.

---

## 2. Операции над матрицами

## 2.1 Сложение матриц

**Теория:**

Матрицы `A` и `B` можно сложить только при одинаковой размерности `m × n`:

\[
(A + B)_{ij} = a_{ij} + b_{ij}
\]

**Код:** `add_matrices(a, b)`:
1. Проверяет размерности через `_shape`.
2. Проверяет равенство размеров.
3. Выполняет поэлементное сложение двойным списковым выражением.
4. Возвращает `CalculationResult(steps, result)`.

## 2.2 Вычитание матриц

**Теория:**

\[
(A - B)_{ij} = a_{ij} - b_{ij}
\]

условие размерностей то же, что и для сложения.

**Код:** `subtract_matrices(a, b)` реализует аналогичный шаблон проверок и поэлементного вычисления.

## 2.3 Умножение матрицы на скаляр

**Теория:**

\[
(kA)_{ij} = k \cdot a_{ij}
\]

**Код:** `multiply_by_scalar(a, scalar)` умножает каждый элемент строки на `scalar`.

## 2.4 Умножение матриц

**Теория:**

Если `A` размера `m × n`, `B` размера `n × p`, то

\[
C = AB,\quad c_{ij} = \sum_{k=1}^{n} a_{ik}b_{kj}
\]

Условие существования: `cols(A) == rows(B)`.

**Код:** `multiply_matrices(a, b)`:
1. Проверка совместимости размерностей.
2. Три вложенных цикла (по `i`, `j`, `k` через comprehension для произведений).
3. Суммирование произведений в `result[i][j]`.
4. Запись подробного шага для каждого элемента `C[i,j]`.

## 2.5 Транспонирование

**Теория:**

\[
(A^T)_{ij} = a_{ji}
\]

Размер `m × n` становится `n × m`.

**Код:** `transpose(a)` переставляет индексы: внешний цикл по столбцам исходной матрицы, внутренний — по строкам.

## 3. Определитель

## 3.1 Теория

Определитель определён только для квадратной матрицы `n × n`.

В проекте используется вариант **метода Гаусса**:

1. Преобразуем матрицу элементарными преобразованиями строк до верхнетреугольной.
2. Определитель верхнетреугольной матрицы равен произведению диагональных элементов.
3. Перестановка двух строк меняет знак определителя.

Идея формально:

\[
\det(A) = (-1)^s \prod_{i=1}^{n} u_{ii}
\]

где `s` — число перестановок строк, `U` — верхнетреугольная форма.

## 3.2 Код

`determinant(a)`:
- проверяет квадратность;
- в каждом столбце выбирает ведущий элемент (pivot) по модулю;
- при обмене строк умножает `det` на `-1`;
- умножает накопленный `det` на текущий ведущий элемент;
- зануляет элементы ниже pivot.

Если pivot практически нулевой (`< 1e-12`) — возвращает `0.0`.

---

## 4. Ранг матрицы

## 4.1 Теория

Ранг — максимальное число линейно независимых строк (или столбцов), равный числу ненулевых строк в ступенчатой форме.

Алгоритм:
1. По столбцам ищем pivot.
2. Нормируем pivot-строку.
3. Обнуляем элементы в pivot-столбце у остальных строк.
4. Количество найденных pivot = ранг.

## 4.2 Код

`rank(a)` реализует приведение к редуцированной ступенчатой форме (почти RREF):
- переменная `r` хранит текущий номер опорной строки;
- поиск pivot в текущем столбце;
- перестановка строк;
- нормализация pivot-строки;
- исключение по столбцу для остальных строк;
- инкремент `r`.

Возвращаемое значение `result` — найденный ранг.

## 5. Решение СЛАУ

Рассматривается система:

\[
Ax = b
\]

где `A` — квадратная матрица `n × n`, `x` и `b` — векторы длины `n`.

## 5.1 Матричный метод

### Теория

При `det(A) \neq 0` матрица обратима:

\[
x = A^{-1}b
\]

### Код

`solve_slae_matrix_method(a, b)`:
1. Проверяет квадратность и размер `b`.
2. Считает `det(A)` и проверяет невырожденность.
3. Строит `A^{-1}` через `_inverse(a)`.
4. Вычисляет `x` как матрично-векторное произведение.

`_inverse(a)` реализован методом Гаусса-Жордана для расширенной матрицы `[A | I]`.

## 5.2 Метод Крамера

### Теория

Если `det(A) \neq 0`, то

\[
x_i = \frac{\det(A_i)}{\det(A)}
\]

где `A_i` — матрица `A`, в которой `i`-й столбец заменён на `b`.

### Код

`solve_slae_cramer(a, b)`:
1. Проверяет предусловия.
2. Считает `det(A)`.
3. Для каждого столбца строит `A_i`, считает `det(A_i)`.
4. Добавляет `x_i = det(A_i)/det(A)`.

## 5.3 Метод Гаусса

### Теория

Система приводится к верхнетреугольному виду (прямой ход), затем решается обратной подстановкой.

Прямой ход:

\[
R_j \leftarrow R_j - \frac{a_{j i}}{a_{i i}} R_i
\]

Обратный ход:

\[
x_i = \frac{1}{a_{ii}}\left(b_i - \sum_{j=i+1}^{n} a_{ij}x_j\right)
\]

### Код

`solve_slae_gauss(a, b)`:
1. Формирует расширенную матрицу `[A|b]`.
2. Выбирает pivot в столбце (частичный выбор).
3. Выполняет прямой ход исключения.
4. Выполняет обратный ход и вычисляет `x`.

## 6. Ошибки и математическая корректность

## 6.1 Теоретически невозможные операции

Примеры:
- сложение матриц разных размеров;
- определитель неквадратной матрицы;
- Крамер при `det(A)=0`.

**Код:** все такие ситуации поднимают `MatrixError` с человекочитаемым текстом.

## 6.2 Устойчивость и пороги

Для проверки нуля используется порог `1e-12`, что учитывает вычисления с плавающей точкой и предотвращает ложные деления на очень малые числа.

---

## 7. Привязка теории к UI

`app.py` реализует связку «ввод → вычисление → вывод»:

1. `parse_matrix` / `parse_vector` читают текст в структуры Python.
2. `do_*`-обработчики вызывают конкретные функции ядра.
3. `show(...)` печатает шаги (`steps`) и итог (`result`).
4. `show_error(...)` выводит диагностическое сообщение.

Таким образом теория (в `matrix_calculator.py`) напрямую отображается в пользовательский сценарий.

---

## 8. Карта соответствия: формула → функция

- `C_{ij}=a_{ij}+b_{ij}` → `add_matrices`
- `C_{ij}=a_{ij}-b_{ij}` → `subtract_matrices`
- `(kA)_{ij}=k a_{ij}` → `multiply_by_scalar`
- `c_{ij}=\sum_k a_{ik}b_{kj}` → `multiply_matrices`
- `(A^T)_{ij}=a_{ji}` → `transpose`
- `det(A)` через Гаусс → `determinant`
- `rank(A)` через ступенчатую форму → `rank`
- `x=A^{-1}b` → `solve_slae_matrix_method` + `_inverse`
- `x_i=det(A_i)/det(A)` → `solve_slae_cramer`
- Гаусс (прямой+обратный ход) → `solve_slae_gauss`
